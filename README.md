# SNHU-CS-330
## Comp Graphic and Visualization

In this corse, we were tasked with creating an OpenGL solution that rendered a real world 3D scene.  I was tasked with taking a picture of anything (my desk) and replicating it using multiple shapes to simulate complex objects.  We gave the scene colors, textures, materials, and lighting.  We also implemented movement around the scene so we could view the objects we were designing.  The solution was a visual studio solution file in C and C++.

### Things about design that I learned in this course:
- Objects overlaping can create complex object without requiring complex shapes.
- You can make almost anything from a small number of shapes.
- OpenGL or any rendering API pipline can be as simple or as complex as you make it.
- Understanding OpenGL design gives you an open framework to apply the same principles to any future work.

### Things about developing programs that I learned in this course:
- This project helped me to better hone my organization skills such as using arrays for complex objects.
- Over the course of this project (which we have been working on since the second week) I learned to put more emphasis on README files for version control (what have I done, what do I need to do).
- I have always taken great pains to use proper white space, however, scrolling through 1000+ lines showed me that other techniques are often needed.
- Developed my ability to design programs that increase in scale.
- Reminded me the importance K.I.S.S. and D.R.Y. (Keep It Simple Stupid and Don't Repeat Yourself).
- Develop code that can be replicated for use in future projects/solutions (as general as possible).

Computer science has honestly changed the way I approach problems. Before I started this degree, I would look at something complicated and feel overwhelmed. Now, I automatically start breaking things down into smaller pieces and thinking through them step by step. That mindset has helped me not just in programming, but in other classes and even everyday situations. It’s given me confidence because I know that if I keep working through a problem logically, I’ll eventually figure it out.

Working with computational graphics and visualization has been one of the biggest growth areas for me. When I build a 3D scene or adjust lighting and camera angles, I’m actually applying math concepts like vectors and transformations without just memorizing formulas. I can see the results instantly on the screen, which makes everything feel more real and easier to understand. It’s one thing to solve equations on paper, but it’s completely different to watch those equations control how objects move, rotate, and interact in a scene. That experience has helped me connect abstract math concepts to something practical.

Graphics programming has also forced me to think more deeply about how different parts of a system work together. If something renders incorrectly, the issue could be in my object transformations, my lighting calculations, my shader code, or even how data is being sent to the GPU. Tracking down those issues has made me more patient and more detail-oriented. It’s taught me that even small mistakes can cause big problems, and that careful testing and debugging are just part of the process.
